**Краткое описание проекта**  
RESTful API для банковской системы с управлением пользователями, картами и переводами. Реализована полноценная система аутентификации с JWT токенами, ролевой доступ и операции с банковскими картами.

**Аутентификация и токены**  
Подход к аутентификации  
Используется статистический JWT подход с двумя типами токенов:

**Access Token**  
Назначение: Доступ к защищенным endpoint'ам

- Срок жизни: 1 час (3600000 ms)
- Хранение: Redis (для возможности
  отзыва)
- Структура: Содержит roles пользователя


**Refresh Token**

- Назначение: Обновление access token'а
-    Срок жизни: 7 дней
     (604800000 ms)
- Хранение: Redis
   - Структура: Только базовая
     информация


**Flow аутентификации**

- Логин → Возвращает access + refresh tokens
-       Доступ к API → Access
  token в header Authorization: Bearer <token>

- Истечение access token → Использование refresh token для  
  получения нового
-  Логаут→ Удаление токенов из Redis



**После запуска**  
Документация API  
Swagger UI: http://localhost:8080/swagger-ui.html

OpenAPI: http://localhost:8080/v3/api-docs

Тестовые пользователи  
При первом запуске автоматически создается:

Admin: admin@example.com / admin123

Роли: ADMIN (полный доступ)

**Основные endpoints  
Аутентификация**
- POST /api/v1/auth/register - Регистрация

- POST /api/v1/auth/login - Вход

- POST /api/v1/auth/refresh - Обновление токена

- GET /api/v1/auth/logout - Выход

**Карты (требуют аутентификации)**
- POST /api/v1/cards - Создание карты (ADMIN only)

- GET /api/v1/cards/my - Мои карты

- POST /api/v1/cards/transfer - Перевод между картами

- PATCH /api/v1/cards/{id}/block - Блокировка карты

**Пользователи**
- PATCH /api/v1/users/me - Обновление профиля

- POST /api/v1/users/me/password - Смена пароля
